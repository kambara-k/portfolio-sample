Springの基本構成とアノテーション

:contentReference[oaicite:0]{index=0} はJavaでよく使われるフレームワークで、 特に :contentReference[oaicite:1]{index=1} を使うことで簡単にWebアプリを開発できる。 主にDI（依存性注入）とアノテーションベースの開発が特徴である。 

						<-- 依存性注入（DI） -->
						@Component
						class SampleService {
						    public void hello() {
						        System.out.println("Hello Spring");
						    }
						}
						
						<-- Controller（画面やAPIの入口） -->
						@RestController
						@RequestMapping("/api")
						class SampleController {
						
						    private final SampleService service;
						
						    public SampleController(SampleService service) {
						        this.service = service;
						    }
						
						    @GetMapping("/hello")
						    public String hello() {
						        return service.hello();
						    }
						}
						
						<-- Service（ビジネスロジック） -->
						@Service
						class SampleService {
						    public String hello() {
						        return "Hello Spring";
						    }
						}

Springでは、役割ごとにクラスを分けることで保守性を高める。 また、アノテーションを付けることでSpringが自動的に管理（DI）を行う。 

						<-- よく使うアノテーション -->
						@Component      // 一般的なコンポーネント
						@Service        // ビジネスロジック層
						@Repository     // データアクセス層
						@RestController // REST API用コントローラ
						@RequestMapping // URLルーティング
						@GetMapping     // GETリクエスト
						@PostMapping    // POSTリクエスト
```
